# 001 — Мультимодульный монорепозиторий

## Контекст
Проект состоит из пяти сервисов. Нужно решить: один репозиторий или больше.

## Решение
Мультимодульный Gradle-проект в одном репозитории.

## Обоснование
- Много репозиториев при одном разработчике создают много проблем
- Общие DTO (события Kafka) лежат в модуле common — не нужен отдельный репозиторий
- Одна команда собирает всё

## Альтернативы
- Разные репозитория + common-lib в Maven Central — избыточно для одного разработчика
- Один монолитный модуль — нет явного разделения ответственности
