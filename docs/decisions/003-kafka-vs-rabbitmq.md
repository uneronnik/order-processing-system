# 003 — Kafka вместо RabbitMQ

## Контекст
Нужен брокер сообщений для асинхронной коммуникации между сервисами.

## Решение
Apache Kafka.

## Обоснование
- Kafka хранит сообщения на диске — можно перечитать историю
- Consumer groups обеспечивают масштабирование обработки
- Широко используется в банках и финтехе

## Альтернативы
- RabbitMQ — проще в настройке, но удаляет сообщения после доставки.
