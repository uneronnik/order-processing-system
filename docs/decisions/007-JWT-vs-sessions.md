# 007 — JWT вместо серверных сессий

## Контекст
Нужен механизм аутентификации пользователей, который работает в микросервисной архитектуре.

## Решение
Stateless JWT-токены. Auth-service генерирует токен, остальные сервисы проверяют подпись с тем же секретом.

## Обоснование
- Stateless — сервисам не нужно общее хранилище сессий (Redis и т.д.)
- Каждый сервис самостоятельно валидирует токен без обращения к auth-service
- Горизонтальное масштабирование без sticky sessions
- Токен содержит email и роль — не нужен дополнительный запрос за данными пользователя

## Trade-off
- Токен нельзя отозвать до истечения срока (нет серверного состояния)
- JWT-секрет должен быть одинаковым во всех сервисах — утечка компрометирует всю систему
- Размер токена больше, чем session ID в cookie

## Альтернативы
- Серверные сессии + Redis — позволяют отзывать сессии, но требуют общий Redis и добавляют точку отказа
- OAuth2 / OpenID Connect — правильный выбор для продакшена, но избыточен для этого проекта
