# 005 — Strategy-паттерн для уведомлений

## Контекст
Notification Service должен отправлять уведомления через разные каналы (лог, Telegram, в будущем — email, SMS).

## Решение
Интерфейс NotificationSender + реализации для каждого канала. NotificationRouter получает List<NotificationSender> через DI и вызывает все активные каналы.

## Обоснование
- Добавление нового канала — один новый класс, без изменения существующего кода (Open/Closed Principle)
- @ConditionalOnProperty позволяет включать/выключать каналы через конфигурацию
- Каждый канал изолирован — ошибка в Telegram не ломает логирование

## Альтернативы
- if/else или switch по типу канала — нарушает OCP, растёт с каждым новым каналом
- Отдельный микросервис на каждый канал — избыточно для текущего масштаба
