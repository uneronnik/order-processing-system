# 002 — Testcontainers вместо H2

## Контекст
Нужна стратегия для интеграционных тестов с базой данных.

## Решение
Testcontainers с реальным PostgreSQL.

## Обоснование
- H2 не поддерживает все фичи PostgreSQL (например, gen_random_uuid())
- Тесты на H2 могут проходить, а на PostgreSQL — ломаться
- Testcontainers поднимает настоящий PostgreSQL в Docker за секунды

## Trade-off
- Тесты немного медленнее (старт контейнера ~3 сек)
- Требуется Docker на машине разработчика
